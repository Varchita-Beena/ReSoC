# ReSoC
Large language models (LLMs) can answer questions and often provide a step-by-step explanation, called a chain-of-thought (CoT). However, these explanations are not always trustworthy: a model may first decide the answer and then write an explanation afterward. This creates a gap between what the model says and how it actually reached the answer. In this project, we study post-hoc reasoning, meaning explanations written after the final answer is produced. Our goal is not to improve explanations, but to check whether they are faithful, that is, whether the explanation truly supports the model’s answer. We evaluate our method on five benchmark datasets: StrategyQA, OpenBookQA, GSM8K, WinoBias, and Tell2Design. We collect model answers and explanations, and we also use human annotation to judge whether explanations are correct, relevant, and consistent with the final answer.

We propose **Re**SoC: **Re**verse **So**cratic **C**hain-of-Thought, a reverse question reconstruction test for rationale faithfulness. Given a rationale $r$ for a question $q$, we generate candidate questions $q'$ from $r$ and re-evaluate the model; a rationale is treated as faithful if the regenerated instance recovers the ground-truth answer. We additionally compute cosine similarity between $(q, q')$ and between $(r, r')$ to measure consistency. For each sample, we generate three candidate questions and select the one with the highest similarity. We test corruption sensitivity: we slightly edit the explanation (for example, changing a number or adding/removing “not”) and run the same pipeline again. If these small changes do not affect the reconstructed question or the final answer or the reasoning, it suggests the explanation may not be truly connected to the model’s decision.
